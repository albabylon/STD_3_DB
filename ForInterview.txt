1 Что такое ADO.NET?
Это внутренняя часть Microsoft .NET Framework, которая обеспечивает доступ к XML, реляционным БД, различным типам данных приложений

2 
DataSet — это объект, представляющий собой автономную архитектуру, 
которая может сохранять содержимое таблицы 
и может содержать более одной таблицы из одного и того же источника данных.

DataReader — это объект, представляющий собой подключенную архитектуру, 
которая не может сохранять содержимое таблицы, 
поскольку обеспечивает доступ к данным только для пересылки и только для чтения.


3 Что такое объект подключения?
Вспоминаем про то, как происходит подключение базы данных в C#. 
Различные объекты подключения используются для разных поставщиков, 
таких как объект OleDbConnection для поставщика OLE-DB и объект SqlConnection для сервера Microsoft SQL.

4 Чем отличается LEFT JOIN от RIGHT JOIN?
LEFT JOIN показывает все записи из левой таблицы (той, которая идет первой в join-выражении) независимо от наличия соответствующих записей в правой таблице.
RIGHT JOIN действует в противоположно LEFT JOIN — показывает все записи из правой (второй) таблицы и только совпавшие из левой (первой) таблицы.

5 Для чего нужны индексы в базе данных?
Индексы необходимы для оптимизации поиска по БД. Простым языком: индексы ускоряют поиск информации.





Entity Framework — это высокоуровневая надстройка над ADO.NET и провайдерами данных, 
которая позволяет работать с БД в объектно-ориентированном стиле, минимизируя ручное написание SQL.

Если ADO.NET — это "ручное управление" (как механик, который сам собирает двигатель), то EF — это "автопилот" 
(вы просто говорите, куда ехать, а система сама управляет двигателем).

БД — библиотека с книгами.

СУБД — библиотекарь, который управляет книгами.

Провайдер — переводчик, который объясняет библиотекарю, что вам нужно.

ADO.NET — набор инструментов (каталог, форма запроса), чтобы работать с библиотекой через переводчика.
